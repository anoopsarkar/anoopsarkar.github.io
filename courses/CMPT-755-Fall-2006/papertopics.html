Final Paper Topics for CMPT-755

One of these topics can be a valid paper topic for this course. If you have your own topic that you wish to pursue (this is encouraged!), please provide a one-page written proposal to me and I will discuss it with you. Topics can be either the theoretical analysis of some technique useful for compilers or an implementation and comparison between alternate methods. Implementations can use existing tools available on the web.
  1. Implement and test 2 different edit-distance based syntax error correction algorithms: either 2 different CFG algorithms or compare regular languages and CFL-based algorithms.
  2. Provide a theoretical analysis or an implementation of code generation from parse trees using tree matching by exploiting the notion of tree transducers.
    • H. Comon et al. Tree Automata Techniques and Applications. http://www.grappa.univ-lille3.fr/tata
    • J.W. Thatcher, Tree automata: an informal survey, Currents in the Theory of Computing (Alfred V.Aho, ed.), chapter 4, pp. 143--172, Prentice-Hall, 1973.
  3. Invent a new algorithm or analyze an existing algorithm that takes a set of input strings and produces the smallest CFG that generates exactly this set. One idea is to use block edit distance to improve on current smallest grammar algorithms.
    • Eric Lehman. Approximation Algorithms for Grammar-Based Data Compression. PhD thesis, 2002.
    • Also cf. references in above thesis.
  4. Provide an algorithm which takes as input a parse tree and a deterministic CFG and enumerates all the input strings that when parsed with the CFG will result in the parse tree. Such an algorithm has important applications in designing sequences that lead to a particular structure in the fields of computational linguistics and biological sequence analysis.
  5. Review existing Monte Carlo and Las Vegas randomized algorithms for recognition of regular and context-free languages. Can we use a Monte Carlo algorithm that uses regular language recognition algorithms and uses it for context-free recognition, or similarly uses context-free algorithms for context-sensitive recognition?
  6. We know the answer to the question: is there an algorithm which can decide if a particular language represented by a CFG in the set of regular language. This is possible when the CFG is equivalent to a deterministic PDA. As far as I know it does not hold for arbitrary CFGs (due to emptiness?). It is known that there is a doubly exponential upper and lower bound to this algorithm. Can this algorithm be extended to other sub-classes of CFGs, e.g. regular closures over deterministic CFGs.
    • Stearns: A regularity test for pushdown machines, Information and Control 11: 3, 323-340, 1967.
    • Valiant: Regularity and related problems for deterministic pushdown automata, JACM 22: 1, 1-10, 1975.
    • E. Bertsch and M.-J. Nederhof. Regular closure of deterministic languages. SIAM Journal on Computing, 29(1):81-102, 1999.
  7. If we have G1 and G2 and they are both regular grammars then it easy to show if L(G1) = L(G2). The following result extends this for the languages generated by two deterministic pushdown automata. A review of this result would be acceptable as a final paper.
    • Geraud Senizergues, L(A)=L(B)? Decidability results from complete formal systems, Theoretical Computer Science 251, pp.1-166, 2001.
